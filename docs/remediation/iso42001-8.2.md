# Every production model has linked dataset documentation

`ISO42001-8.2` · framework **iso42001** · severity **high** · Do

## What this control checks

ISO 42001 §8.2 (and EU AI Act Article 10) require providers to
document the data sets used for training, validation, and testing.
Concord enforces this by requiring every production model in the
registry to carry a `dataset_card_url` MLflow tag pointing at a
retrievable dataset documentation artifact.

## Why it matters

The dataset is half the model — its provenance, licensing, known
biases, and version determine whether the model is fit for its
declared purpose. Auditors cannot accept a model into a high-risk
production environment without documented data lineage and
quality criteria, and ISO/EU make this explicit.

## Evidence

Collected from the `mlflow` source (`model_registry` evidence type).

## What a failure looks like

This control reports a finding when any of the following hold:

- no model registry evidence collected
- production model <value> has no dataset_card_url tag
- production model <value> has a dataset_card_url but no dataset_version tag (recommended)
- high-risk model <value> has no dataset_quality_assessment tag (required for EU AI Act Annex IV §2)

## Remediation

Bring each affected resource or attestation listed under *What a failure looks like* into compliance, then re-collect evidence. Estimated effort: **3h**. Automated fix available: **false**.

## How to re-verify

```
concord check --controls <pack>/controls --framework iso42001 --control-id ISO42001-8.2
```

A passing run reports this control green; in CI, `concord gate` exits non-zero while it fails.
