# Every production AI model has a fresh evaluation report on file

`ISO42001-7.4` · framework **iso42001** · severity **high** · Do

## What this control checks

ISO 42001 §7.4 requires AI systems to be continuously evaluated
against measurable performance criteria after deployment. Concord
enforces this by requiring every production model in the registry
to carry an evaluation_report tag (pointing at the artifact) and a
last_evaluated_at tag within the configurable freshness window.

## Why it matters

Static evaluation at deployment is not enough — model performance
drifts as input distributions change. Without periodic
re-evaluation, the organization cannot evidence that the deployed
model still meets its declared performance and fairness criteria,
which directly maps to EU AI Act Article 17 (Quality Management
System) for high-risk AI systems.

## Evidence

Collected from the `mlflow` source (`model_registry` evidence type).

## What a failure looks like

This control reports a finding when any of the following hold:

- no model registry evidence collected
- production model <value> has no evaluation_report tag
- production model <value> has no last_evaluated_at tag
- production model <value> was last evaluated over <value> days ago
- high-risk model <value> has no owner tag

## Remediation

Bring each affected resource or attestation listed under *What a failure looks like* into compliance, then re-collect evidence. Estimated effort: **2h**. Automated fix available: **false**.

## How to re-verify

```
concord check --controls <pack>/controls --framework iso42001 --control-id ISO42001-7.4
```

A passing run reports this control green; in CI, `concord gate` exits non-zero while it fails.
